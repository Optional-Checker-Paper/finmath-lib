15    /**
16    * This class implements a stochastic Levenberg Marquardt non-linear least-squares fit
17    * algorithm.
18    * <p>
19    * The design avoids the need to define the objective function as a
20    * separate class. The objective function is defined by overriding a class
21    * method, see the sample code below.
22    * </p>
23    *
24    * <p>
25    * The Levenberg-Marquardt solver is implemented in using multi-threading.
26    * The calculation of the derivatives (in case a specific implementation of
27    * {@code setDerivatives(RandomVariable[] parameters, RandomVariable[][] derivatives)} is not
28    * provided) may be performed in parallel by setting the parameter <code>numberOfThreads</code>.
29    * </p>
30    *
31    * <p>
32    * To use the solver inherit from it and implement the objective function as
33    * {@code setValues(RandomVariable[] parameters, RandomVariable[] values)} where values has
34    * to be set to the value of the objective functions for the given parameters.
35    * <br>
36    * You may also provide an a derivative for your objective function by
37    * additionally overriding the function {@code setDerivatives(RandomVariable[] parameters, RandomVariable[][] derivatives)},
38    * otherwise the solver will calculate the derivative via finite differences.
39    * </p>
40    * <p>
41    * To reject a point, it is allowed to set an element of <code>values</code> to {@link java.lang.Double#NaN}
42    * in the implementation of {@code setValues(RandomVariable[] parameters, RandomVariable[] values)}.
43    * Put differently: The solver handles NaN values in <code>values</code> as an error larger than
44    * the current one (regardless of the current error) and rejects the point.
45    * <br>
46    * Note, however, that is is an error if the initial parameter guess results in an NaN value.
47    * That is, the solver should be initialized with an initial parameter in an admissible region.
48    * </p>
49    *
50    * The following simple example finds a solution for the equation <br>
51    * <table>
52    * <caption>Sample linear system of equations.</caption>
53    * <tr><td>
54    * 0.0 * x<sub>1</sub> + 1.0 * x<sub>2</sub> = 5.0
55    * </td></tr>
56    * <tr><td>
57    * 2.0 * x<sub>1</sub> + 1.0 * x<sub>2</sub> = 10.0
58    * </td></tr>
59    * </table>
60    *
61    * <pre>
62    * <code>
63    * LevenbergMarquardt optimizer = new LevenbergMarquardt() {
64    * // Override your objective function here
65    * public void setValues(RandomVariable[] parameters, RandomVariable[] values) {
66    * values[0] = parameters[0] * 0.0 + parameters[1];
67    * values[1] = parameters[0] * 2.0 + parameters[1];
68    * }
69    * };
70    *
71    * // Set solver parameters
72    * optimizer.setInitialParameters(new RandomVariable[] { 0, 0 });
73    * optimizer.setWeights(new RandomVariable[] { 1, 1 });
74    * optimizer.setMaxIteration(100);
75    * optimizer.setTargetValues(new RandomVariable[] { 5, 10 });
76    *
77    * optimizer.run();
78    *
79    * RandomVariable[] bestParameters = optimizer.getBestFitParameters();
80    * </code>
81    * </pre>
82    *
83    * See the example in the main method below.
84    *
85    * <p>
86    * The class can be initialized to use a multi-threaded valuation. If initialized
87    * this way the implementation of <code>setValues</code> must be thread-safe.
88    * The solver will evaluate the gradient of the value vector in parallel, i.e.,
89    * use as many threads as the number of parameters.
90    * </p>
91    *
92    * Note: Iteration steps will be logged (java.util.logging) with LogLevel.FINE
93    *
94    * @author Christian Fries
95    * @version 1.6
96    */
