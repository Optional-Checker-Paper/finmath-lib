18    /**
19    * This class implements a multi-asset Black Scholes model providing an <code>AbstractProcessModel</code>.
20    * The class can be used with an EulerSchemeFromProcessModel to create a Monte-Carlo simulation.
21    *
22    * The model can be specified by general factor loadings, that is, in the form
23    * \[
24    * dS_{i} = r S_{i} dt + S_{i} \sum_{j=0}^{m-1} f{i,j} dW_{j}, \quad S_{i}(0) = S_{i,0},
25    * \]
26    * \[
27    * dN = r N dt, \quad N(0) = N_{0}.
28    * \]
29    *
30    * Alternatively, the model can be specifies by providing volatilities and correlations
31    * from which the factor loadings \( f_{i,j} \) are derived such that
32    * \[
33    * \sum_{k=0}^{m-1} f{i,k} f{j,k} = \sigma_{i} \sigma_{j} \rho_{i,j}
34    * \]
35    * such that the effective model is
36    * \[
37    * dS_{i} = r S_{i} dt + \sigma_{i} S_{i} dW_{i}, \quad S_{i}(0) = S_{i,0},
38    * \]
39    * \[
40    * dN = r N dt, \quad N(0) = N_{0},
41    * \]
42    * \[
43    * dW_{i} dW_{j} = \rho_{i,j} dt,
44    * \]
45    * Note that in case the model is used with an EulerSchemeFromProcessModel, the BrownianMotion used can
46    * have a correlation, which alters the simulation (which is admissible).
47    * The specification above hold, provided that the BrownianMotion used has independent components.
48    *
49    * The class provides the model of \( S_{i} \) to an <code>{@link net.finmath.montecarlo.process.MonteCarloProcess}</code> via the specification of
50    * \( f = exp \), \( \mu_{i} = r - \frac{1}{2} \sigma_{i}^2 \), \( \lambda_{i,j} = \sigma_{i} g_{i,j} \), i.e.,
51    * of the SDE
52    * \[
53    * dX_{i} = \mu_{i} dt + \sum_{j=0}^{m-1} \lambda_{i,j} dW_{j}, \quad X_{i}(0) = \log(S_{i,0}),
54    * \]
55    * with \( S = f(X) \). See {@link net.finmath.montecarlo.process.MonteCarloProcess} for the notation.
56    *
57    * @author Christian Fries
58    * @see net.finmath.montecarlo.process.MonteCarloProcess The interface for numerical schemes.
59    * @see net.finmath.montecarlo.model.ProcessModel The interface for models provinding parameters to numerical schemes.
60    * @version 1.1
61    */
