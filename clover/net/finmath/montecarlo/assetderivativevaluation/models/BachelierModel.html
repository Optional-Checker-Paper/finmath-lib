17    /**
18    * This class implements a (variant of the) Bachelier model, that is,
19    * it provides the drift and volatility specification
20    * and performs the calculation of the numeraire (consistent with the dynamics, i.e. the drift).
21    *
22    * The model is
23    * \[
24    * d(S/N) = \sigma dW, \quad S(0) = S_{0},
25    * \]
26    * \[
27    * dN = r N dt, \quad N(0) = N_{0},
28    * \]
29    *
30    * Note: This implies the dynamic
31    * \[
32    * dS = r S dt + \sigma exp(r t) dW, \quad S(0) = S_{0},
33    * \]
34    * for \( S \). For The model
35    * \[
36    * dS = r S dt + \sigma dW, \quad S(0) = S_{0},
37    * \]
38    * see {@link net.finmath.montecarlo.assetderivativevaluation.models.InhomogenousBachelierModel}.
39    *
40    * The model's implied Bachelier volatility for a given maturity T is
41    * <code>volatility * Math.exp(riskFreeRate * optionMaturity)</code>
42    *
43    * The class provides the model of S to an <code>{@link net.finmath.montecarlo.process.MonteCarloProcess}</code> via the specification of
44    * \( f = \text{identity} \), \( \mu = \frac{exp(r \Delta t_{i}) - 1}{\Delta t_{i}} S(t_{i}) \), \( \lambda_{1,1} = \sigma \), i.e.,
45    * of the SDE
46    * \[
47    * dX = \mu dt + \lambda_{1,1} dW, \quad X(0) = \log(S_{0}),
48    * \]
49    * with \( S = X \). See {@link net.finmath.montecarlo.process.MonteCarloProcess} for the notation.
50    *
51    * @author Christian Fries
52    * @version 1.0
53    * @see net.finmath.montecarlo.process.MonteCarloProcess The interface for numerical schemes.
54    * @see net.finmath.montecarlo.model.ProcessModel The interface for models provinding parameters to numerical schemes.
55    */
